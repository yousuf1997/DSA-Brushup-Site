What is PriorityQueue?

A PriorityQueue is as name suggest that the elements will be organzied depends on the priority of the elmement. Internally it maintains heap, and there are two types of heap min and max. The top value of min heap wil always smaller than the rest of the values in the queue and max is vice versa. By default it orders the elment its according the natural ordering of the object usually this is done using compareTo() internally.

How to initialize PriorityQueue?

PriorityQueue<Integer> pq = new PriorityQueue<Integer>();

How to add elements to the PriorityQueue?

pq.add(34);
pq.add(42);
pq.add(11);
pq.add(97);
pq.add(87);
Time complexity : O(log n)

The elments above will be stored in sorted order (default : ascending) despite the insertion order. Sorted order : 11, 34, 42, 87, 97. As you can see by default its using min heap.

How to check if an element is present in the PriorityQueue?

pq.contains(97);
Time complexity : O(n)

How to get the top element without removing it?

If its a max queue, the following will return the max element in the queue otherwise min element.

pq.peek();

How to get the top element and also remove it?

If its a max queue, the following will return the max element in the queue otherwise min element.

pq.pop();
Time complexity : O(log n)

How to iterate the PriorityQueue and remove the elements?

while(!queue.isEmpty()){
  System.out.println(pq.poll());
}
Time complexity : O(n)

If you like to order element based on some other sorting logic you can implement compartor interface. Lets says that we want to order the element by descending order.

You have to return the following values to define the ordering.
  • Return positive value (1) to state that the first paramenter is bigger than the second.
  • Return negative value (-1) to state that the first paramenter is smaller than the second.
  • Return 0 value to state that both values are equal.

public class DescendingComparator implements Comparator<Integer> {
  @Override
  public int compare(Integer o1, Integer o2) {
       return o1 - o2;
  }
}
When you initialize the PriorityQueue you pass the compartor to the constructor as follows.

PriorityQueue<Integer> pq = new PriorityQueue<Integer>( new DescendingComparator() );
If you are using an existing type such as Integer or String, you can use this simple way to order the elements in reverse order in priority queue. To initiate the reverse order behavior during the initialization of the priority queue pass the first parameter takes the initial capacity, and the second is the rever order compartor. Default initial capactiy of the PriorityQueue is 11.

PriorityQueue<Integer> pq = new PriorityQueue<Integer>(11, Collections.reverseOrder());
